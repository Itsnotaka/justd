## Alert Dialog
When dealing with an **alert** dialog, there are two key things to nail down. First, make sure the modal doesn't close when tapping outside of it. Second, hide the close button unless it's provided. The simple fix? Just set the `isDismissable` prop on the modal, and youâ€™re good to go.
<How toUse="overlays/alert-dialog-demo" />
If you're not using the `Modal` component directly but still want to manage the modal experience, switch gears and use the `ModalOverlay`. Just pass the `isDismissable` prop to it, and you're all set.

```tsx
<ModalOverlay isDismissable={false}/>
```

It might be a good idea to extract the modal into a separate component for better organization.

```
interface ModalActionProps {
  title: string
  description: string
  confirmText: string
  action: () => void
  closeModal: () => void
  loading: boolean
  isOpen: boolean
}

export function ModalAction(props: ModalActionProps) {
  return (
    <ModalOverlay isOpen={props.isOpen} onOpenChange={props.closeModal}>
      <ModalContent>
        <ModalHeader>
          <ModalTitle>{props.title}</ModalTitle>
          <ModalDescription>{props.description}</ModalDescription>
        </ModalHeader>
        <ModalFooter>
          <ModalClose>Cancel</ModalClose>
          <Button className="min-w-24" isDisabled={props.loading} onPress={props.action}>
            {props.loading ? <LoadingDots className={'bg-fg'} /> : props.confirmText}
          </Button>
        </ModalFooter>
      </ModalContent>
    </ModalOverlay>
  )
}
```

Then you can use it like this.

```
<ModalAction {...actionType(state)} closeModal={closeModal} loading={loading} isOpen={!!state} />
```

With that, now we can modify the `actionType` function to return the initial state.

```tsx
const actionType = (t: string | null) => {
  let initialsState = {
    title: '',
    description: '',
    confirmText: '',
    action: () => {}
  }

  switch (t) {
    case 'delete': ...
    case 'ban': ...
    case 'restore': ...
    default:
      return initialsState
  }
}
```
